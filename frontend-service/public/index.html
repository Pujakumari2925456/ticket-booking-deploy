Design Decisions
CACHE-ASIDE PATTERN
Redis stores inventory in RAM. Reads take ~1ms vs ~20ms from disk. Prevents race conditions atomically via DECR.
EVENT-DRIVEN ASYNC
RabbitMQ absorbs 10K req/sec as a shock absorber. Returns 202 instantly. Worker persists at sustainable pace.
DISTRIBUTED RATE LIMITING
Rate limits stored in Redis, not memory. Works across multiple gateway instances. Blocks bots at edge.
EVENTUAL CONSISTENCY
Redis is source of truth during sale. MongoDB catches up asynchronously. Dead letter queue handles failures.
